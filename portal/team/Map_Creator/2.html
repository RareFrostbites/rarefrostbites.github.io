If the Map contains a resource/ behavior packs, what's the proper manifest file should be inside?

Manifest's data on a Map's resource/ behavior pack file sometime's messy if we use with a pack builder like Add-on Maker, Texture Maker, etc. We will teach you how to organize them, having an organized manifest's data is great, easy to see and edit while you interact to it.
This is your main Map's file contents, and it has 2 pack folders, resource_packs and behavior_packs, in this lesson, if you didn't see a screenshot photo for behavior_packs, don't worry it has a same process. First thing to do is:

1.

Locate the resource/ behavior_packs file and find manifest.json


Find the pack folder you installed on your Map and then their manifest.json file.
Looks pretty messy though? To fix this messy manifest data, go to 3t3 Minecraft Manifest Data Generator..



2.

Enter the required text


Select the Pack type into Resource, put the title what your Map's title on the textbox and the full description of the Map.
NOTE: Always include Read Rules and Regulations by §3Rare§bFrostbites §l§2Team after the last description.



3.

Generate it


Generate it, make another one for Behavior pack, the process are still the same.
Copy the generated text inside and copy to the manifest file. If you copied the generated text with a pack file type "resource", then paste it inside manifest.js on Your Map's resource pack folder, if you copied the generated text with a pack file type "behavior", then paste it inside manifest.js on Your Map's behavior pack folder.
See? Looks organized, so anyone who will see this makes easy to edit.
