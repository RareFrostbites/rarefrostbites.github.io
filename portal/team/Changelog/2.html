Manage Changelogs file(s)

Part 2

How to manage for Beta Pack(s)?

If you made changes to a Pack and it's beta published, you need to make a changelogs file first.
Here's the steps how:

1.

Make a new folder


Create a folder inside the changelogs folder and name it into beta.

2.

Put a new file for a new changelogs


You can put one but every one version only, don't make one and one more for every changes, if so, put all you made the changes in one file before you publish and for the new version, make another one.

3.

Make sure it's eligible


Don't

Don't make it an introduction style inside the changelogs file

Do

Make it simple, only a list(s) of the changes you made
NOTE: If you're making a changelog(s) with Mash-up Pack, kindly do your changelog(s) into the Map package.
